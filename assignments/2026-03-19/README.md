## 🔥 중요 질문

### 💡 콜백과 고차 함수의 차이는?

- 고차 함수: 고차 함수란 함수를 인자로 받거나, 함수를 리턴값으로 반환하는 함수.

- 콜백 함수: 인자로써 다른 함수에 전달되는 함수

- 콜백 함수는 고차 함수에게 호출됨

<br>

### 💡 콜백 지옥이 문제가 되는 이유

- 콜백 함수를 중첩해서 사용하면 코드의 가독성이 떨어지고, 흐름을 파악하기 어렵다.

- `Promise`, `async/await` 같은 방법들을 사용 가능

<br>

### 💡 Promise는 콜백과 무엇이 다른가요?

- 콜백은 함수를 인자로 전달해 비동기 작업이 끝난 후 실행하는 방식이고,

- 중첩이 많아질 경우 콜백 지옥으로 인해 가독성이 떨어질 수 있다.

- 반면 Promise는 비동기 작업의 상태와 결과를 객체로 관리하며,

- then/catch를 통해 체이닝 방식으로 작성할 수 있어 가독성과 에러 처리를 개선할 수 있다.

<br>

### 💡 `setTimeout(..., 0)`은 정확히 즉시 실행인가요?

즉시 실행되지 않는다.

- 자바스크립트에서 비동기 함수는 이벤트 루프에 의해 처리되며, 이벤트 큐에 들어간 뒤 콜 스택이 비었을 때 실행된다.

- 따라서 지연 시간이 0이라도 현재 콜 스택에 있는 코드가 모두 끝난 후에 실행된다.

<br>

### 💡 `for`문 안의 비동기 코드에서 `var`가 왜 문제인가요?

- var는 블록 스코프가 아닌 함수 스코프를 가진다.

- 따라서 각 반복 실행마다 새로운 변수가 생성되는 것이 아닌 하나의 var 변수이며,

- 비동기 함수의 실행 시점은 for문이 모두 실행된 이후이다.

- 따라서 각 반복마다 다른 값이 할당 되는 것이 아닌 for문이 모두 실행된 시점의 var 변수의 값을
- 모든 비동기 함수의 클로저가 참조하게 된다.

<br>

### 💡 이벤트 핸들러도 콜백인가요?

이벤트 핸들러도 콜백 함수이다.

```javaScript
button.addEventListener('click', handler);

button.addEventListener('click', function() {
    console.log("클릭 발생!");
});
```

위 코드에서 `function () { ... }`는 브라우저가 클릭 이벤트 발생 시

나중에 실행되도록 하는 함수를 정의한 것이다.

콜백 함수란 인자로써 다른 함수에 전달되는 함수이며, `handler` 역시 인자로

전달되므로 이벤트 핸들러도 콜백이라고 볼 수 있다.

<br>

### 💡 `map`, `filter`, `forEach`의 차이는?

1️⃣ `map`

- 배열의 요소들을 꺼내어 정의한 함수에 따라 새로운 배열로 변환
- 원본 배열은 변경하지 않음
- 입력 배열의 길이 == 출력 배열의 길이

```javaScript
const numbers = [1, 2, 3];
const double = numbers.map(n => n * 2); // [2, 4, 6]
```

<br>

2️⃣ `filter`

- 콜백 함수가 `true`를 반환하는 요소들만 모아 새로운 배열을 만든다.
- 원본 배열은 변경하지 않음
- 입력 배열의 길이 >= 출력 배열의 길이

```javaScript
const numbers = [1, 26, 3, 5, 66];
const overTen = numbers.filter(n => n > 10); // [26, 66]
```

<br>

3️⃣ `foreach`

- 배열의 각 요소를 순회하며 특정 동작을 수행
- 리턴값 없음

```javaScript
const numbers = [1, 2, 3];

numbers.forEach(n => console.log(n)); // 출력: 1 2 3
```
